# Устранение неполадок

## Деплой: `git merge --ff-only` не прошёл

На сервере локальная ветка разошлась с `origin/main`. Нужно вручную синхронизировать (merge/rebase) или исправить состояние репозитория на сервере, затем повторить деплой.

## Backend не видит `TELEGRAM_BOT_TOKEN`

1. Проверить Secret **`TELEGRAM_BOT_TOKEN`** в GitHub.
2. На сервере: в корне репо файл **`.env`** с строкой `TELEGRAM_BOT_TOKEN=...`.
3. Пересоздать контейнер backend: `docker compose up -d --force-recreate backend`.

## API 500 при сохранении пользователей

Смотреть логи backend и таблицу `system_logs`. Частые темы: уникальный `login`, архивированные пользователи — логика восстановления по логину в `routers/auth.py`.

## Фронт не бьётся с API локально

- Docker отдаёт API на **8003**, Vite по умолчанию проксирует на **8000** — выровняйте порты (см. `docs/00-ONBOARDING.md`).

## Telegram: 409 Conflict

Два процесса с одним токеном бота. Остановите дубликаты (другой сервер, второй systemd, локальный запуск).

## Nginx: 502 на `/api`

- Backend не запущен или слушает не тот порт.
- Проверить `curl http://127.0.0.1:8003/health` на сервере.

## Миграции

Ошибки при старте API в логах контейнера. Проверить подключение к БД и целостность таблицы `alembic_version`. Ручной откат/правка только осознанно, с бэкапом.
