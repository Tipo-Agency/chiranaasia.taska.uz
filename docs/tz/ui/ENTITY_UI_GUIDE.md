# UI‑гайд по сущностям tipa.taska.uz

## 1. Зачем этот документ

Цель: **зафиксировать визуальные паттерны** для всех основных сущностей, чтобы:

- рабочий стол, модули и настройки выглядели **однородно**;
- новые экраны делались по тем же правилам;
- не появлялись “четвёртые” стили вкладок, карточек и таблиц.

Документ описывает:

- какие компоненты отвечают за каждую сущность;
- какие **UI‑паттерны** использовать (карточки, таблицы, табы, модалки);
- как должны выглядеть заголовки, кнопки действий и фильтры.

> Фронтенд расположен в `apps/web`. Ниже указаны конкретные компоненты, с которыми нужно синхронно работать.

---

## 2. Базовые паттерны интерфейса

### 2.1. Заголовки страниц

- Компонент: `AppHeader`.
- Всегда показывает:
  - иконку и название текущего раздела;
  - глобальный поиск;
  - переключатель темы, уведомления, меню пользователя.
- Текущие названия:
  - `Рабочий стол`, `Задачи`, `Воронка продаж`, `Клиенты`, `Финансовое планирование`, `Бизнес‑процессы`, `Встречи`, `Документы`, `Склад`, `Сайты`, `Сотрудники`, `Аналитика`, `Настройки`.

### 2.2. Навигация слева (основное меню)

- Компонент: `Sidebar`.
- Это **единственное** левое меню приложения:
  - пункты: Рабочий стол, Задачи, Воронка, Клиенты, Склад, Финансы, Бизнес‑процессы, Встречи, Документы, Сотрудники, Аналитика, Контент‑планы, Беклог, Функционал, Сайты, Настройки.
- Внутри разделов **не дублируем** полноценные сайдбары — только лёгкие табы/фильтры в контенте.

### 2.3. Таб‑навигация (вкладки)

Единый паттерн “пилюль”:

- Используется в:
  - `HomePage` — Входящие / Исходящие / Сообщения (рабочий стол);
  - `SettingsView` — табы настроек (горизонтальная навигация);
  - `ArchiveView` — типы сущностей в архиве;
  - `AutomationSettings` — выбор модуля (Задачи / Документы / Встречи / …).
- Вёрстка:
  - скруглённая “пилюля”, размер `text-xs | text-sm`;
  - активное состояние: тёмный фон, белый текст (или наоборот в dark‑режиме);
  - не больше **1 строки табов** — при избытке включается горизонтальный скролл.

### 2.4. Карточки и списки

- Списки сущностей по умолчанию — **карточки в колонку**:
  - фон: `bg-white dark:bg-[#252525]`;
  - рамка: `border border-gray-200 dark:border-[#333]`;
  - закругление: `rounded-xl`;
  - отступы: `p-4` / `p-5`;
  - hover: лёгкая тень и подсветка рамки.
- Примеры:
  - пользователи в `ProfileSettings (activeTab="users")`;
  - проекты/модули в `SpaceSettings (activeTab="projects")`;
  - статьи расходов в `FinanceCategoriesSettings`;
  - фонды в `FundsSettings`;
  - воронки в `SalesFunnelsSettings`.

### 2.5. Таблицы

- Когда нужно видеть много записей в плотном виде (финансы, склад) — используем `table`:
  - заголовок в `thead` с фоном `bg-gray-50`;
  - строки `tr` с hover‑подсветкой;
  - действия (иконки редактирования/удаления) выровнены справа.
- Примеры:
  - `FinanceCategoriesSettings` (список категорий);
  - `InventoryView` (таблицы складских остатков, движений).

### 2.6. Модалки

- Общие правила:
  - затемнение фона: `fixed inset-0 bg-black/40 backdrop-blur-sm`;
  - карточка: `max-w-md`/`max-w-lg`, `rounded-xl`, `border`, `shadow-2xl`;
  - заголовок в шапке + кнопка закрытия `X`.
- Примеры:
  - `TaskModal`, `IdeaModal`, `FeatureModal`;
  - модалки настроек (создание категорий, фондов и т.п.).

## 3. Сущности и их UI по модулям

### 3.1. Рабочий стол (`HomePage`)

- Компонент: `apps/web/components/pages/HomePage.tsx`.
- Основные блоки:
  - **Приветствие** (через `HomeHeader`): “Добрый день, {Имя}!” + дата.
  - **Верхняя кнопка** “Создать” с дропдауном: Задача, Сделка, Процесс, Заявка на приобретение.
  - **Карточка “Входящие / Исходящие / Сообщения”**:
    - табы‑пилюли (один стиль с другими модулями);
    - внутри — список сущностей (сейчас заглушка, потом подключение inbox/deals/tasks).
  - **Обзор**:
    - `StatsCards` — ключевые метрики (выручка, план, дебиторка);
    - `MyTasksSection` — мои активные задачи;
    - `NewDealsSection` + `UpcomingMeetings`.

**Важно:** любые новые вкладки/фильтры на рабочем столе должны использовать **тот же стиль пилюль**, что и в `SettingsView`, `AutomationSettings` и `ArchiveView`.

### 3.2. Задачи (`TasksPage`, `TasksView`, `TaskModal`)

- Сущность: `Task`.
- Компоненты:
  - список/доска/гант: `TasksPage` и `TasksView`;
  - просмотр/редактирование: `TaskModal`, `IdeaModal`, `FeatureModal`.
- Паттерн:
  - в списке — карточки с названием, статусом, приоритетом, дедлайном;
  - статусы/приоритеты тянутся из справочников, настраиваемых в `SettingsView` → `Статусы` / `Приоритеты`.

### 3.3. CRM (клиенты, сделки, договоры, дебиторка)

- Сущности:
  - `Client`, `Deal`, `Contract`, `AccountsReceivable`, `SalesFunnel`.
- Компоненты:
  - модуль: `CRMModule` (`view === 'sales-funnel'`);
  - отдельная страница по клиентам/договорам: `ClientsPage`.
- UI‑паттерн:
  - **воронка**: колонки‑стадии, внутри карточки сделок;
  - карточки сделок и клиентов — тот же стиль, что и на рабочих/настройках (белые рамки, иконки статусов);
  - создание/редактирование через модалки.

### 3.4. Финансы (`FinanceModule`, настройки)

- Сущности:
  - `FinanceCategory`, `Fund`, `FinancePlan`, `PurchaseRequest`.
- Компоненты:
  - модуль: `FinanceModule` (`view === 'finance'`);
  - настройки:
    - `FinanceCategoriesSettings` — список категорий с типом (фикс/процент);
    - `FundsSettings` — фонды.
- Паттерн:
  - списки категорий/фондов — табличный вид;
  - создание/редактирование — модалки с короткими формами;
  - в аналитике и рабочем столе используются агрегированные карточки из тех же сущностей.

### 3.5. HR и бизнес‑процессы (`HRModule`)

- Сущности:
  - `EmployeeInfo`, `Department`, `OrgPosition`, `BusinessProcess`.
- Компоненты:
  - `HRModule` (`view === 'employees'` / `view === 'business-processes'`);
  - `DepartmentsView` — настройка подразделений;
  - `ArchiveView` — архив сотрудников/подразделений/процессов.
- Паттерн:
  - сотрудники — карточки с аватаром, именем, логином, действиями;
  - департаменты — таблица/карточки с названиями и количеством сотрудников.

### 3.6. Встречи (`MeetingsModule`)

- Сущность: `Meeting`.
- Компоненты:
  - `MeetingsModule` (`view === 'meetings'`);
  - связанные настройки (напоминания/уведомления) — во вкладке Automation (`AutomationSettings`, модуль `meetings`).
- Паттерн:
  - список встреч — карточки с датой, временем, участниками;
  - создание/редактирование — модалка.

### 3.7. Документы (`DocumentsModule`, `DocEditor`)

- Сущности:
  - `Doc`, `Folder`.
- Компоненты:
  - `DocumentsModule` (`view === 'docs'`);
  - `DocEditor` (`view === 'doc-editor'`).
- Паттерн:
  - список документов — таблица/карточки (название, теги, тип);
  - отдельная страница редактора без сайдбара, с собственным хедером.

### 3.8. Склад (`InventoryView`)

- Сущности:
  - `Warehouse`, `InventoryItem`, `StockBalance`, `StockMovement`, `InventoryRevision`.
- Компонент:
  - `InventoryView` (`view === 'inventory'`).
- Паттерн:
  - табы‑пилюли по разделам (Склады / Товары / Остатки / Движения / Инвентаризации);
  - каждый раздел — таблица с действиями справа;
  - модалки для добавления/редактирования.

### 3.9. Настройки (`SettingsView` и под‑компоненты)

- Главная сущность: **“конфигурация системы”**, разбитая на:
  - профиль + пользователи (`ProfileSettings`);
  - проекты/страницы/статусы/приоритеты (`SpaceSettings`);
  - департаменты (`DepartmentsView`);
  - финансы (`FinanceCategoriesSettings`, `FundsSettings`);
  - CRM‑воронки (`SalesFunnelsSettings`);
  - уведомления и роботы (`AutomationSettings`);
  - интеграции (Telegram, сайт, Meta) (`IntegrationSettings`);
  - архив (`ArchiveView`).
- Паттерн:
  - сверху — заголовок “Настройки системы” + кнопка “Закрыть”;
  - под ним — **единая горизонтальная навигация** табами‑пилюлями (без второго бокового меню);
  - ниже — контент выбранного раздела (карточки/таблицы/формы по стандартным правилам).

## 4. Правила для будущих экранов

1. **Не добавлять новые боковые меню.**  
   У приложения уже есть глобальный `Sidebar`. Внутри разделов:
   - только горизонтальные табы‑пилюли;
   - или лёгкие подзаголовки и фильтры.

2. **Для вкладок всегда использовать один и тот же стиль.**  
   Настройки, рабочий стол, автоматизация, архив — все табы должны визуально совпадать.

3. **Карточки и таблицы — только из описанных шаблонов.**  
   Если нужен новый тип списка, сначала добавляем его сюда как паттерн, затем реализуем.

4. **Все новые сущности привязывать к существующим компонентам.**  
   Например, новый тип задачи → `TasksView` + те же модалки; новая сущность CRM → тот же стиль карточки, что у сделок/клиентов.

5. **Визуальная консистентность важнее “красоты отдельного экрана”.**  
   Если новый экран требует отличный от остальных UI‑паттерн, нужно сначала описать его здесь и согласовать, а уже потом реализовывать.
