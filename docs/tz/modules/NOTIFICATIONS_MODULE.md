# Техническое задание: Модуль уведомлений (Notifications Module)

## 1. Общая информация

### 1.1. Назначение
Модуль уведомлений предназначен для централизованной отправки уведомлений пользователям через различные каналы (Activity Logs, Telegram).

### 1.2. Принципы работы
- **Activity Logs** - Всегда создаются для всех пользователей
- **Telegram** - Отправляются только по настройкам пользователя

## 2. Каналы уведомлений

### 2.1. Activity Logs (Системные уведомления)

#### 2.1.1. Всегда включены
- Уведомления в системе всегда создаются для всех пользователей
- Отображаются в разделе "Уведомления"
- Фильтрация по пользователю, типу действия, дате

#### 2.1.2. Типы уведомлений
- Создание задачи
- Изменение статуса задачи
- Назначение задачи
- Комментарий к задаче
- Создание сделки
- Изменение стадии сделки
- И другие события системы

### 2.2. Telegram уведомления

#### 2.2.1. Настройки
- **telegramPersonal** (boolean) - Уведомления в личный чат
- **telegramGroup** (boolean) - Уведомления в группу

#### 2.2.2. Получатели
- Личный чат: пользователь с заполненным `telegramUserId`
- Группа: чат с ID из `telegramGroupChatId`

## 3. Типы событий

### 3.1. Задачи (Tasks)
- **newTask** - Новая задача
- **statusChange** - Изменение статуса
- **taskAssigned** - Назначение задачи
- **taskComment** - Комментарий к задаче
- **taskDeadline** - Приближение дедлайна

### 3.2. CRM
- **dealCreated** - Новая сделка
- **dealStatusChanged** - Изменение стадии сделки
- **clientCreated** - Новый клиент
- **contractCreated** - Новый договор

### 3.3. Контент
- **docCreated** - Новый документ
- **meetingCreated** - Новая встреча

### 3.4. Финансы
- **purchaseRequestCreated** - Новая заявка на покупку

## 4. Централизованный сервис

### 4.1. notificationService.ts

#### 4.1.1. Функции
- `notifyTaskCreated` - Уведомление о создании задачи
- `notifyTaskStatusChanged` - Уведомление об изменении статуса
- `notifyDealCreated` - Уведомление о создании сделки
- `notifyDealStatusChanged` - Уведомление об изменении стадии
- `notifyClientCreated` - Уведомление о создании клиента
- `notifyContractCreated` - Уведомление о создании договора
- `notifyDocCreated` - Уведомление о создании документа
- `notifyMeetingCreated` - Уведомление о создании встречи
- `notifyPurchaseRequestCreated` - Уведомление о создании заявки

#### 4.1.2. Параметры
```typescript
{
  context: {
    currentUser: User;
    allUsers: User[];
    notificationPrefs?: NotificationPreferences;
  }
}
```

### 4.2. activityLogUtils.ts

#### 4.2.1. Функции
- `createActivityLog` - Создать лог активности
- `createActivityLogsForAll` - Создать логи для всех пользователей
- `createTaskCreatedLog` - Лог создания задачи
- `createTaskStatusChangedLog` - Лог изменения статуса
- И другие специализированные функции

### 4.3. telegramService.ts

#### 4.3.1. Функции
- `sendTelegramNotification` - Отправить уведомление в Telegram
- `getUserTelegramChatId` - Получить ID чата пользователя
- `sendTelegramMessage` - Отправить сообщение в Telegram

## 5. Форматирование сообщений

### 5.1. Форматы сообщений
- HTML форматирование для Telegram
- Эмодзи для визуального выделения
- Кнопки для действий

### 5.2. Примеры сообщений
- "✅ Создана задача: [Название]"
- "🔄 Изменен статус задачи: [Статус] → [Новый статус]"
- "💰 Новая сделка: [Название] на сумму [Сумма]"

## 6. Интеграция

### 6.1. Интеграция с модулями
- Все модули используют централизованный сервис уведомлений
- Единый интерфейс для всех типов уведомлений
- Централизованная обработка ошибок

### 6.2. Интеграция с Telegram ботом
- Использование того же API для отправки сообщений
- Синхронизация настроек уведомлений

## 7. Технические детали

### 7.1. Обработка ошибок
- Все ошибки логируются с префиксом `[NOTIFICATION]`
- Ошибки не прерывают выполнение основной логики
- Единый стиль обработки ошибок

### 7.2. Производительность
- Асинхронная отправка уведомлений
- Не блокирует основную логику
- Очередь для массовых уведомлений

### 7.3. Таблицы в базе данных
- `activity` - Логи активности (уведомления в системе)
