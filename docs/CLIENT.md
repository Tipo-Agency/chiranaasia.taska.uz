# Система для заказчика — развёртывание и эксплуатация

Кратко, без внутренней архитектуры репозитория.

## Что входит в поставку

- Веб-приложение (браузер), API и база данных PostgreSQL.
- Опционально: Telegram-бот (отдельный сервис на сервере).

## Окружение

- Сервер с **Docker** (PostgreSQL и backend в контейнерах).
- **nginx** для HTTPS и раздачи собранного фронтенда.
- **Node.js** на сервере или в CI — только для сборки фронта при деплое.

## Деплой

- Обновления кода: **push в ветку `main`** → автоматический деплой (GitHub Actions → скрипт на сервере).
- На сервере задаются секреты (SSH, токен Telegram-бота и т.д.) — см. корневой `README.md` в репозитории и `docs/operations/DEPLOY.md`.

## После установки

- Проверка: сайт открывается по HTTPS, `GET /health` возвращает успешный статус.
- Пользователей и права настраивают через интерфейс **Настройки** (или первичное создание администратора по процедуре заказчика).

## Резервное копирование

- Регулярный **дамп PostgreSQL** (volume Docker) — зона ответственности эксплуатации.
- Секреты и `.env` не хранить в открытом виде в репозитории.

## Поддержка

- Логи ошибок приложения: раздел настроек / админ-панель (по ролям).
- Технические вопросы по хостингу и домену — к команде, сопровождающей сервер.
