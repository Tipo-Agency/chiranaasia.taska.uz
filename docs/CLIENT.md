# Кратко для заказчика / эксплуатации

Документ без внутренних деталей репозитория — что нужно для стабильной работы **tipa.taska.uz** как продукта.

## Что это

Веб-приложение для задач, CRM (воронка, лиды, клиенты), внутренних коммуникаций, финансового планирования и связанных процессов. Доступ через браузер; данные хранятся на сервере заказчика.

## Развёртывание

- Сервер с **Docker**, **nginx**, при необходимости **Node.js** только для **сборки** фронтенда при обновлении.
- Автодеплой из ветки `main` через GitHub Actions (при настроенных секретах).
- После выкладки проверяются: открытие сайта, `**GET /health`**, работа входа, при необходимости — логи в разделе настроек приложения.

## Эксплуатация

- Администрирование пользователей, справочников, воронок и т.д. — через раздел **«Настройки»** в приложении.
- Прямой доступ к PostgreSQL (pgAdmin и т.п.) — **технический** инструмент для разработчика, не замена админки.
- Резервное копирование БД — по политике заказчика; рекомендуется регулярный дамп Postgres.

## Поддержка и интеграции

- Входящие лиды с сайта — по API с ключом из настроек воронки (документация для интегратора — у команды разработки / внутренняя `docs/API.md`).
- Telegram / Instagram — токены и ключи задаются в настройках и на сервере; не передаются клиентским JS в открытом виде для критичных операций.

## Контакты

Технические вопросы — к команде, сопровождающей репозиторий и сервер.
