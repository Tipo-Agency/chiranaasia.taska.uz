# Сущности (предметная область)

Здесь — **человекочитаемые** описания сущностей: зачем нужны, основные поля, связи. Точные типы колонок и ограничения — в **SQLAlchemy-моделях** и миграциях.

## Как вести документы

- Имя файла: **`kebab-case.md`** (`weekly-plan.md`, `content-post.md`).
- Для новой сущности скопируйте **`_TEMPLATE.md`**.
- При изменении бизнес-логики обновляйте соответствующий файл и **`docs/architecture/DATABASE.md`** (если менялась таблица).
