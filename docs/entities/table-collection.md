# Коллекция таблиц (`TableCollection`)

## Назначение

Настраиваемая «таблица» в системе: **контент-план**, бэклог, функциональность и т.д. Определяет колонки, статусы и контекст для задач и постов.

## Бизнес-правила

- У таблицы есть **тип** (например `content-plan`), от него зависит UI (календарь, канбан).
- Поддерживается **архивация** (`is_archived`).
- Публичный просмотр контент-плана — по `table_id` (см. `GET /tables/public/content-plan/{table_id}`).

## Хранение в БД

**Таблица:** `tables`  
**Модель:** `TableCollection` в `apps/api/app/models/settings.py`.

## API

`apps/api/app/routers/tables.py` — в т.ч. публичный эндпоинт для контент-плана.

## UI

Модуль пространств, `TableView`, `ContentPlanView`, публичная страница `PublicContentPlanView`.
