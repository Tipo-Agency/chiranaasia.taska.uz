# Пользователь (`User`)

## Назначение

Учётная запись для входа в систему, привязка к роли и данным сотрудника в интерфейсе.

## Бизнес-правила

- Вход по **логину** и **паролю**; пароль в БД хранится как **bcrypt hash**.
- Возможна **архивация** (`is_archived`); при создании пользователя с логином существующего архивного — запись может **восстанавливаться** (см. `auth.update_users`).
- Роль **ADMIN** открывает API `/api/admin/*` и экран админ-панели.

## Хранение в БД

**Таблица:** `users`  
**Модель:** `apps/api/app/models/user.py`

Основные поля (точный список — в модели): идентификатор, логин, хеш пароля, имя, роль, флаги архива и активности.

## Связи

Используется во всех доменах как `userId` / `assigneeId` и т.д.

## API

Роутер **`apps/api/app/routers/auth.py`**: логин, список пользователей, обновление (в т.ч. создание).
