# Задача (`Task`) и проект (`Project`)

## Назначение

**Проект** — логическая группировка работ.  
**Задача** — единица работы с дедлайном, статусом, приоритетом, исполнителем.

## Бизнес-правила

- Статусы и приоритеты — из справочников (`statuses`, `priorities`), привязанных к контексту таблиц/проекта (см. фронт и роутеры `statuses` / `priorities`).
- Задачи отображаются в модуле задач, на рабочем столе, в поиске, в интеграциях с ботом.

## Хранение в БД

| Таблица | Модель |
|---------|--------|
| `projects` | `Project` |
| `tasks` | `Task` |

Файл: `apps/api/app/models/task.py`.

## Связи

- Задача ссылается на проект, исполнителей, таблицу-источник (в зависимости от сценария) — детали в модели.

## API

`apps/api/app/routers/tasks.py`, `projects.py`.
