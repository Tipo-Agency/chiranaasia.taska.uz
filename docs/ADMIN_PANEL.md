# Админ-панель

Отдельный раздел для администраторов: просмотр таблиц БД, логов ошибок, метрик нагрузки и запуск автотестов.

## Доступ

- В боковом меню пункт **«Админ-панель»** отображается только у пользователей с ролью **ADMIN**.
- Все запросы к API админки (`/api/admin/*`) требуют:
  1. Вход по логину и паролю (чтобы в сессии был JWT).
  2. Роль пользователя **ADMIN** (проверяется на бэкенде).

Если войти только по выбору пользователя из списка (без пароля), JWT не сохраняется — запросы к админке вернут 401. Нужно выйти и войти заново, введя логин и пароль администратора.

## Разделы админки

1. **База данных** — список таблиц БД и просмотр данных с пагинацией (только чтение).
2. **Ошибки** — записи из `system_logs` (ERROR, CRITICAL, WARNING) с фильтром по уровню.
3. **Нагрузка** — health API, размер БД, количество записей по таблицам.
4. **Тесты** — кнопка «Запустить тесты» выполняет `pytest apps/api/tests` и выводит результат в браузере.

## API (бэкенд)

- `GET /api/admin/tables` — список таблиц и количество записей.
- `GET /api/admin/tables/{name}?offset=0&limit=100` — данные таблицы (пагинация).
- `GET /api/admin/health` — состояние API и БД.
- `GET /api/admin/stats` — статистика по таблицам и размер БД.
- `POST /api/admin/tests/run` — запуск pytest, возвращает вывод и код выхода.

Все эти маршруты защищены зависимостью `get_current_user_admin` (JWT + роль ADMIN).

## Логи и ошибки

Системные ошибки по-прежнему доступны в **Настройки → Система / Логи** (без обязательного JWT). В админке вкладка «Ошибки» дублирует эти данные и дополнительно требует права администратора для единообразия доступа к админ-разделу.
