# Как добавить новую сущность (чеклист)

Универсальный порядок, чтобы фронт, бэк и БД оставались согласованными.

## 1. Модель и миграция

1. Добавить SQLAlchemy-модель в **`apps/api/app/models/<domain>.py`** (или новый файл + импорт в **`models/__init__.py`**).
2. Создать миграцию Alembic: `alembic revision --autogenerate -m "add_<table>"`.
3. Проверить сгенерированный файл в **`alembic/versions/`**, при необходимости поправить вручную.
4. `alembic upgrade head` локально.

## 2. API

1. Добавить роутер или расширить существующий в **`apps/api/app/routers/`**.
2. Подключить роутер в **`app/main.py`**.
3. Сериализация: единый стиль полей (часто **camelCase** в JSON для фронта — смотрите соседние роутеры).

## 3. Фронтенд

1. Типы в **`apps/web/types.ts`** (или сгенерировать OpenAPI при необходимости).
2. Методы в **`apps/web/services/apiClient.ts`** (или `backend/api.ts`).
3. Загрузка/сохранение в соответствующем слайсе **`frontend/hooks/slices/`** и **`useAppLogic.ts`** (подключение к загрузке модулей).
4. UI: новый компонент или вкладка; прокинуть пропсы через **`*Module.tsx`** / **`AppRouter.tsx`**.

## 4. Документация

1. Добавить строку в **`docs/architecture/DATABASE.md`** (таблица, назначение).
2. Создать или обновить файл в **`docs/entities/`** по шаблону **`_TEMPLATE.md`**.

## 5. Архивация

Если сущность «удаляется» из UI — предпочитайте флаг **`is_archived`** (или согласованное имя), а не физическое удаление, если так принято для домена.

## 6. Права доступа

При необходимости: проверка роли в зависимостях FastAPI (`get_current_user`, `get_current_user_admin`).
